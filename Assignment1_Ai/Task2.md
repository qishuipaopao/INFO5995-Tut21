Summarise：
This Android application is a user authentication system that allows users to register, log in, and access protected features. The application authenticates users using a username and password and generates a session token upon login to maintain user state. Key features include: user registration, user login, basic user interface interaction, and user logout.



System Model:

[ User ]
   ↓
[ Android App UI ]
   ↓
[ Login/Register Logic ]
   ↓
[ Token Generation (Random Value) ]
   ↓
[ Basic Operation Interface ]
   ↓
[ Log Out ]

User Interface (UI): Allows users to register and log in using a username and password.
Credential Storage: User credentials are stored locally in a file (credentials.txt).
Session Management: Upon successful login, a session token is generated and stored in SharedPreferences.
Protected Activity: The Profile activity is accessible after session creation.

Data Flow
User inputs username and password.
The application verifies credentials from a local file.
If valid, a session token is generated.
The token is stored locally and used to indicate authentication.



Core Assumptions:

1. The attacker can decompile the APK.

2. The attacker can observe and understand application behavior (decompiled code).

3. The attacker can attempt to log in multiple times.

4. The attacker can read or predict the "random values" generated by the application.

Attacker's Goal: To obtain other users' session tokens and impersonate accounts.
